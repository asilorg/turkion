## Заголовки

Используйте заголовки для введения основных разделов. Они структурируют документацию и помогают пользователям ориентироваться в содержимом.

::code-preview
---
class: "[&>div]:*:my-0"
---
## Заголовки

#code
```mdc
## Заголовки
```
::

### Подзаголовки

Используйте подзаголовки для дальнейшего деления разделов. Они создают более детальную иерархию содержимого для лучшей читаемости.

::code-preview
---
class: "[&>div]:*:my-0"
---
### Подзаголовки

#code
```mdc
### Подзаголовки
```
::

::tip
Каждый заголовок и подзаголовок создаёт якорь и автоматически отображается в оглавлении.
::

## Форматирование текста

Nuxt UI поддерживает большинство параметров форматирования Markdown.

| Стиль       | Использование | Результат  |
| ----------- | ------------- | ---------- |
| Жирный      | `**bold**`    | **Жирный** |
| Курсив      | `*italic*`    | *Курсив*   |
| Зачёркнутый | `~~strike~~`  | ~~Зачёркнутый~~ |

Комбинируйте форматирование для создания более богатых стилей текста.

| Стиль                  | Использование       | Результат         |
| ---------------------- | ------------------- | ----------------- |
| Жирный курсив          | `**_bold italic_**` | ***Жирный курсив*** |
| Жирный зачёркнутый     | `~~**bold**~~`      | ~~**Жирный**~~    |
| Курсив зачёркнутый     | `~~*italic*~~`      | ~~*Курсив*~~      |

Для степеней, индексов или математических обозначений используйте HTML-теги `<sup>` и `<sub>`.

| Стиль      | Использование            | Результат   |
| ---------- | ------------------------ | ----------- |
| Надстрочный | `<sup>superscript</sup>` | superscript |
| Подстрочный | `<sub>subscript</sub>`   | subscript   |

## Ссылки

Ссылки соединяют различные части документации и внешние ресурсы, необходимы для навигации пользователей.
Чтобы создать ссылку, заключите текст ссылки в скобки `[]()`.

::code-preview
---
class: "[&>div]:*:my-0"
---
[Nuxt UI](https://ui.nuxt.com/getting-started/installation)

#code
```mdc
[Nuxt UI](https://ui.nuxt.com/getting-started/installation)
```
::

### Внутренние ссылки

Для ссылок внутри документации используйте пути от корня, например `/getting-started/installation`.

::code-preview
---
class: "[&>div]:*:my-0"
---
[Установка](/getting-started/installation)

#code
```mdc
[Установка](/getting-started/installation)
```
::

## Списки

Организуйте связанные элементы в структурированном, удобочитаемом формате. Markdown поддерживает неупорядоченные, упорядоченные и вложенные списки.

### Неупорядоченный

Используйте неупорядоченные списки для элементов без определённой последовательности. Начинайте каждый элемент с символа `-`.

::code-preview
---
class: "[&>div]:*:my-0"
---
- Я элемент списка.
- Я ещё один элемент списка.
- Я последний элемент списка.

#code
```mdc
- Я элемент списка.
- Я ещё один элемент списка.
- Я последний элемент списка.
```
::

### Упорядоченный

Используйте упорядоченные списки, когда важен порядок элементов, например в пошаговых инструкциях. Начинайте каждый элемент с числа.

::code-preview
---
class: "[&>div]:*:my-0"
---
1. Я элемент списка.
2. Я ещё один элемент списка.
3. Я последний элемент списка.

#code
```mdc
1. Я элемент списка.
2. Я ещё один элемент списка.
3. Я последний элемент списка.
```
::

### Вложенный

Создавайте иерархические списки с вложенными элементами для сложных структур. Отступ вложенных элементов — четыре пробела.

::code-preview
---
class: "[&>div]:*:my-0"
---
- Я элемент списка.
  - Я вложенный элемент списка.
  - Я ещё один вложенный элемент.
- Я ещё один элемент списка.

#code
```mdc
- Я элемент списка.
  - Я вложенный элемент списка.
  - Я ещё один вложенный элемент.
- Я ещё один элемент списка.
```
::

## Таблицы

Представляйте структурированные данные в строках и столбцах. Таблицы идеально подходят для сравнения данных или перечисления свойств.

::code-preview
---
class: "[&>div]:*:my-0 [&>div]:*:w-full"
---
| Prop    | Default   | Type     |
| ------- | --------- | -------- |
| `name`  |           | `string` |
| `size`  | `md`      | `string` |
| `color` | `neutral` | `string` |

#code
```mdc
| Prop    | Default   | Type                     |
|---------|-----------|--------------------------|
| `name`  |           | `string`{lang="ts-type"} |
| `size`  | `md`      | `string`{lang="ts-type"} |
| `color` | `neutral` | `string`{lang="ts-type"} |
```
::

## Блок-цитаты

Выделяйте важные цитаты, ссылки или акцентированный текст. Блок-цитаты визуально отделяют цитируемое содержимое.

### Однострочная

Однострочные блок-цитаты лучше всего подходят для коротких, ёмких цитат. Чтобы создать блок-цитату, добавьте `>` перед абзацем.

::code-preview
---
class: "[&>div]:*:my-0"
---
> Nuxt UI — это коллекция Vue-компонентов, composables и утилит для создания красивых и доступных пользовательских интерфейсов.

#code
```mdc
> Nuxt UI — это коллекция Vue-компонентов, composables и утилит для создания красивых и доступных пользовательских интерфейсов.
```
::

### Многострочная

Многострочные блок-цитаты подходят для длинных цитат или когда необходимо включить несколько абзацев в одну цитату.

::code-preview
---
class: "[&>div]:*:my-0"
---
> Nuxt UI — это коллекция Vue-компонентов, composables и утилит для создания красивых и доступных пользовательских интерфейсов.
>
> Создавайте красивые, адаптивные и доступные Vue-приложения с Nuxt UI.

#code
```mdc
> Nuxt UI — это коллекция Vue-компонентов, composables и утилит для создания красивых и доступных пользовательских интерфейсов.
>
> Создавайте красивые, адаптивные и доступные Vue-приложения с Nuxt UI.
```
::
